


观察者模式：
1. 什么时候该考虑此模式：当你(观察者)需要监控(订阅)某个对象(主题)的状态变化，并希望在状态变化的时候能得到通知时。
2. 该模式体现的设计原则：松耦合
3. 注意：观察者模式有"推送"或"拉取"两种模式，大同小异，这里的例子使用的是"推送"方式。两种方式区别在于：
    a.“推送”会一次性把所有数据推送给观察者, 而这些数据可能不全是观察者需要的。
    b.“拉取”方式，则观察者可以通过主题的getter()来获取自己感兴趣的那部分数据。
4. java 有一个自带的观察者模式，分别是下面两个类：
    java.util.Observer (接口)
    java.util.Observable (这是个具体类！！！里面的注册方法是同步的)
    由于后者是一个具体类，java又不能多重继承，所以java自带的观察者模式不是很好用，不能用的时候还是需要自己实现。
