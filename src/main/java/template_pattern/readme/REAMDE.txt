

模板方法模式：
1. 当你发现自己在写一些相似度很高的解决一个问题的步骤(算法)时，可以考虑使用此模式
2. 它在抽象超类中定义了要处理问题的大致共同步骤(框架)，有些步骤是具体的(自己实现)，有些步骤是抽象的(留给子类实现),
    这样就避免了编写大量的重复代码。
3. 它还可以有钩子，钩子是一种被声明在抽象超类中的方法，但是只有空的或者默认的实现，用来干涉上面的“步骤”，
    可以让子类有能力对算法的不同点进行挂钩，要不要挂钩，由子类自行决定。
4. 主要设计原则：好莱坞原则----别调用我(高层组件)，我会调用你(低层组件)
5. 模板模式还有一些其他的变种实现，比如Arrays.sort()方法与Comparable接口的配合就是一种变种，主要是要学习这种超类
    定制好共同的步骤（框架）子类填充各自不同部分的思想。
